# 发布者交互页规格（强制三层输出）

## 硬性前置（每次设计前必须先写）

```
角色锚定：发布者（publisher）— 仅此角色；不混入加入者/运维端
当前页面：{page_route / 中文名}
唯一核心任务（一个动词短语）：{例：发布本周排班任务并开始收集空闲}
非目标（本页明确不做）：{1–3 条}
入口：{从哪来} → 出口：{成功后去哪 / 取消回哪}
```

若用户未给出唯一核心任务，**先问一句再设计**，不得默认「管理一切」。

---

## 强制输出顺序（禁止跳层、禁止合并）

### 第一层：页面布局全局定义

必须包含：

1. **信息架构**：上→下区块列表（每块：名称 / 职责 / 是否可滚动）
2. **焦点**：本页唯一视觉与任务焦点（一句话）
3. **栅格与安全区**：页边距、顶栏、底栏/安全区、主内容区高度策略
4. **主 CTA 位置**：唯一实心主按钮位置；次按钮位置
5. **数据依赖**：首屏请求列表（接口名或资源）、阻塞渲染的关键字段
6. **空 / 载 / 错 三态布局占位**：骨架占哪些块、空态插在哪、错误条位置
7. **边界拦截总表**（页面级）：极端值、权限、乐观锁/version、重复提交

### 第二层：共用组件固定样式参数

只定义**本页用到的**共用件，参数必须可照抄进 WXSS/TDesign/WeUI：

| 组件 | 尺寸/圆角/色/字号/阴影/动画 | 触发方式 | 关闭方式 | z-index/遮罩 |
|------|------------------------------|----------|----------|--------------|
| 弹窗 Dialog | … | … | … | … |
| 抽屉 Drawer | … | … | … | … |
| Toast / Message | … | … | … | … |
| ActionSheet | … | … | … | … |
| 骨架 Skeleton | … | 首屏 loading | n/a | … |
| 空状态 Empty | 插画/文案/按钮 | 列表 length=0 | n/a | … |

颜色/字号必须引用产品 token（`app.wxss` v4）或明确 hex，禁止「淡蓝一点」。

### 第三层：逐按钮完整数据状态机

**每一个可点击控件**（主按钮、次按钮、图标按钮、列表行操作、弹窗确认/取消）单独一节，模板如下：

```markdown
#### btn_{id} · {可见文案}
- 控件类型：button / text-btn / icon / cell
- 默认样式：{token}
- **前置不可用条件**（任一为真则 disabled，并写 disabled 文案/Tooltip）：
  1. …
  2. …
- **可点时：操作触发动作**
  - 前端：校验 → loading 形态（按钮 loading / 全页 / 局部）→ 请求 {API}
  - 防抖/锁：{ms 或 in-flight 锁}
  - 提交 payload 关键字段：{…}
- **分支 A · 后端成功（HTTP/业务 code 成功）**
  - UI 反馈实例：{Toast 文案 / 弹窗关闭 / 按钮恢复}
  - 页面刷新范围：{仅列表局部 | 整页 onShow | 指定字段 setData}
  - 可逆操作入口：{撤销 / 返回编辑 / 无}
  - 导航：{stay | navigateTo | redirectTo | navigateBack}
- **分支 B · 业务逻辑失败（4xx 业务码 / 校验失败）**
  - UI 反馈实例：{字段下红字 / Dialog 标题+正文+按钮}
  - 页面刷新范围：{通常不刷列表；保留用户已填}
  - 可逆操作入口：{改字段后重试 / 联系…}
  - 不离开当前页，除非业务要求
- **分支 C · 网络异常 / 超时 / 5xx**
  - UI 反馈实例：{Toast「网络异常，请重试」或顶部错误条}
  - 页面刷新范围：{不覆盖本地草稿}
  - 可逆操作入口：{同一按钮可再次点击重试}
- **测试用例 ID 建议**：TC-{page}-{btn}-A/B/C
```

## 必须覆盖的横切场景

每个页面规格末尾固定两节：

### 空状态与骨架屏

| 场景 | 触发条件 | UI | 主操作 |
|------|----------|-----|--------|
| 骨架屏 | 首屏请求未返回 | 哪些区块 skeleton | 无 |
| 空列表 | 成功且 length=0 | Empty 文案 | 引导 CTA |
| 空搜索 | 筛选无结果 | 文案+清除筛选 | 清除 |

### 极端边界值拦截

| 字段/动作 | 边界 | 拦截时机 | UI |
|-----------|------|----------|-----|
| 文本长度 | max / 全空格 | blur / submit | 字段错误 |
| 日期 | 结束&lt;开始、跨度过长 | submit | … |
| 重复提交 | 连点 | in-flight disable | 按钮 loading |
| 权限 | 非 publisher | 进页 / 点按钮 | 提示并返回 |
| 并发 | version 冲突 | 接口返回 | Dialog 拉新 |

## 输出格式约定

- 使用 Markdown；第三层每个按钮独立 `####` 标题。
- 文案用**最终产品中文**，可直接当验收文案。
- 每条路径必须能映射为：**Given 前置 / When 点击 / Then UI+数据** 测试用例。
- 不写加入者路径；若按钮对加入者可见，单独标注「本规格不覆盖」或拆另一文档。
- 与 UI 视觉 skill（weui-miniprogram-ui）配合：本 skill 管**交互与状态机**，视觉 token 引用 weui skill / app.wxss。

## 自检清单（交付前）

- [ ] 已写角色=发布者 + 唯一核心任务
- [ ] 三层齐全且顺序正确
- [ ] 每个按钮含：不可用条件 + 触发 + 成功/业务失败/网络 三分支
- [ ] 每分支含：UI 实例、刷新范围、可逆入口
- [ ] 空态骨架 + 边界拦截表
- [ ] 可直接拆 TC-A/B/C
