# 排班小程序 · 增量 PRD（2026-07-21）

> **文档性质**：增量 PRD，仅描述本次 5 项变更，不涉及已有功能完整规格。
> **输出语言**：中文
> **项目类型**：基于已有项目的增量修改（仓库 `D:\排班小程序`，小程序端 `apps/miniprogram/`，后端 `services/api` NestJS）
> **默认技术栈**：微信小程序（WXML/WXSS/JS）+ NestJS 后端；复用 `components/schedule-grid`、`utils/time-format.js`、`domain/date-defaults.js`、`domain/period-builder.js`

---

## 一、产品目标（Product Goals）

| 编号 | 目标 | 说明 |
|------|------|------|
| G1 | 降低任务创建门槛 | 通过「本周/下周」快捷按钮与更合理的布局，减少手动输入与误填（覆盖 7 天上限要求）。 |
| G2 | 提升数据收集灵活性 | 支持手动添加自定义必填项，适配更多填写与收集场景（姓名/学号/联系方式之外的字段）。 |
| G3 | 修复并完善邀请填写分享链路 | 确保分享按钮可达、链接携带邀请标识、按参与方式正确鉴权、链接单次有效且禁止转发。 |

---

## 二、用户故事（User Stories）

| 需求 | 用户故事 |
|------|----------|
| 1. 改名 | 作为创建任务的管理者，我希望把「常见校园起点」这个区块标题改成更通用的「常用作息起点」，避免"校园"场景的局限感。 |
| 2. 本周/下周按钮 | 作为创建任务的管理者，我希望点一下「本周」或「下周」就自动填好这一周的起止日期，而不用一个个翻日历手动选。 |
| 3. 时段规则表格改两列 | 作为创建任务的管理者，我希望「时段规则」区的节次/时段预览以一行两列网格呈现，屏幕更紧凑、一眼看全。 |
| 4. 自定义必填项 | 作为创建任务的管理者，我希望能自己添加必填项（如"宿舍号""班级"），而不仅限于系统下发的姓名/学号/联系方式，且这些自定义项能正常收集与存储。 |
| 5. 微信分享修复 | 作为分组管理者，我希望把填写邀请分享出去后，收到链接的人能凭邀请码直接进入 availability 填写页；且链接只能点开的人用、转发无效，并按"预设名单/所有成员"两种参与方式正确开放权限。 |

## 四、UI 设计稿 / 界面变更描述

### 需求 2：本周/下周 快捷按钮（task-create 第1步 任务信息）
在 dateStart / dateEnd 两个 `picker mode="date"` 上方或之间增加两个快捷按钮：

```
┌─────────────────────────────────────┐
│  任务信息                            │
│                                      │
│  开始日期  [ 2026-07-20 ▼ ]          │  ← picker dateStart
│  结束日期  [ 2026-07-26 ▼ ]          │  ← picker dateEnd
│                                      │
│  [ 本周 ]   [ 下周 ]   ← 新增快捷按钮 │
│   （点击后自动填充周一~周日区间）      │
└─────────────────────────────────────┘
```
> 一周区间恰好 ≤ 7 天，满足 `MAX_TASK_SPAN_DAYS` 拦截规则。

### 需求 3：时段规则 2 列网格（task-create 第2步 时段与规则）
将节次/时段预览区由当前布局改为 2 列卡片网格：

```
┌─────────────────────────────────────┐
│  时段规则                            │
│  [模式 chip] [作息预设] [微调表单]    │
│  ┌──────────┐ ┌──────────┐          │
│  │ 第1节     │ │ 第2节     │  ← 2列  │
│  │ 08:00-    │ │ 09:00-    │          │
│  │ 08:45     │ │ 09:45     │          │
│  └──────────┘ └──────────┘          │
│  ┌──────────┐ ┌──────────┐          │
│  │ 第3节     │ │ 第4节     │          │
│  └──────────┘ └──────────┘          │
└─────────────────────────────────────┘
```

### 需求 4：自定义必填项（task-create 必填信息区）
在固定 chip 下方增加自定义输入行：

```
┌─────────────────────────────────────┐
│  必填信息（可多选）                  │
│  [✓ 姓名] [✓ 学号] [✓ 联系方式]      │  ← 原有 requiredFieldOptions
│  ┌──────────────────────┐ [+ 添加]  │  ← 新增：输入框 + 按钮
│  │ 输入自定义项名，如"宿舍号" │       │
│  └──────────────────────┘           │
│  [✓ 宿舍号]  ← 添加后默认选中        │  ← custom_xxx
└─────────────────────────────────────┘
```
> 提交时 `rules.requiredFields` 除原有 key 外，携带新增的 `custom_*` 项。

### 需求 5：分享入口（task-detail / group-detail，仅调整可见性与 path）
分享按钮位置不变（已存在于 task-detail、group-detail），本次仅：
- 放宽 `wx:if` 可见条件使其在目标角色可达；
- 确保 `path` 携带 `inviteCode`/`shareToken` 指向 `pages/availability`；
- 落地页按"预设名单/所有成员"做登录与权限校验；
- 后端保证 token 一次性、转发失效。

### 需求 1：改名（task-create 第1步区块标题）
仅文本变更：`常见校园起点` → `常用作息起点`（位于作息预设快捷选择区块标题）。
