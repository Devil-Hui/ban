# Identity and Group Operations

## Runtime configuration

All credentials are loaded from environment variables. `.env` is a local ignored file generated by `npm run env:init`; it must never be committed. Production requires `NODE_ENV=production`, `WECHAT_MODE=production`, `WX_APPID`, `WX_SECRET`, a 32-byte hex `PHONE_ENCRYPTION_KEY`, and a non-default `TOKEN_SIGNING_SECRET`.

User JWTs and H5 admin JWTs use separate audiences and cannot be exchanged. Group endpoints accept only user principals. H5 operations may read platform summaries but do not mutate group business records.

## Local scenario seed

Start MySQL, Redis and MinIO, migrate the database, and set `WECHAT_MODE=mock` in the runtime environment:

```bash
npm run infra:up
npm run db:migrate
npm run seed:scenarios
```

The seed is intentionally disabled in production and prints counts only. It uses `mock:U01` through `mock:U13`, is safe to run repeatedly, and never prints tokens, passwords or invite codes.

## Group policy

Owners can manage members, administrators, tasks and dissolution. Administrators can manage members and tasks. Members can view their active groups and manage only their own availability in later scheduling phases. A non-member receives `404` from group routes so group existence is not disclosed; an active member without the required capability receives `403`.

Membership transitions are recorded in `group_member_events` and `audit_logs` within the same transaction. Leaving, kicking and dissolution are soft state changes. A kicked member can rejoin unless `is_blacklisted` is set. Only the owner can remove an administrator or unblock a blacklisted member.
