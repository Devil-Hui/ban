## 一、产品定位

**目标用户**：在校大学生值班排班场景（首期），后续可扩展至个人及企业。

**核心价值**：
- 管理员快速创建排班任务，通过微信邀请码分发
- 加入者像"共享表格"一样标记空闲时间
- 支持上传课表图片 AI 自动识别空闲时段
- 最终智能生成多种排班方案供管理员选择

## 三、核心数据模型

### 3.1 排班分组 (SchedulingGroup)

```
{
  id: string,
  name: string,
  inviteCode: string,        // 6位字母数字邀请码
  createdBy: userId,
  mode: 'timeline' | 'shift' | 'custom',  // 排班模式
  timeConfig: {              // 时间段/节次配置
    periods: [
      { id: 'p1', name: '第一节(8:00-9:40)', start: '08:00', end: '09:40' },
      { id: 'p2', name: '第二节(10:00-11:40)', start: '10:00', end: '11:40' },
      ...
    ],
    dateRange: { start: '2026-09-01', end: '2026-12-31' },
    weekDays: [1,2,3,4,5],    // 星期一~五
  },
  members: [
    { userId, name, avatar, phone, role: 'admin'|'joiner' }
  ],
  status: 'draft' | 'active' | 'completed' | 'archived',
  createdAt, updatedAt
}
```

### 3.2 排班任务 (SchedulingTask)

```
{
  id: string,
  groupId: string,
  title: string,
  description: string,
  dateRange: { start: '2026-09-18', end: '2026-09-24' },  // 本周/下周
  status: 'collecting' | 'completed',  // 正在排班 / 已排班
  template: {                    // 排班样式模板
    style: 1 | 2 | 3,
    periods: ['p1', 'p2', 'p3'], // 适用的节次
  },
  responses: [                   // 加入者的空闲时间标记
    {
      userId: string,
      availableSlots: [
        { date: '2026-09-18', periodId: 'p1' },
        ...
      ],
      source: 'manual' | 'ai_course_table',  // 来源
      submittedAt: timestamp,
    }
  ],
  finalSchedules: [              // 最终排班方案（多方案）
    {
      id: string,
      selected: boolean,
      assignments: [
        { date: '2026-09-18', periodId: 'p1', userId: string, userName: string }
      ]
    }
  ],
  createdAt, updatedAt
}
```

### 3.3 用户课表 (UserCourseTable)

```
{
  userId: string,
  imageUrl: string,            // 原始课表图片
  parsed: {
    semester: '2026-2027学年第1学期',
    courses: [
      {
        name: '高等数学',
        weeks: [1,2,3,...,16],  // 哪些周上课
        dayOfWeek: 1,            // 周一=1
        periods: [1,2],          // 第1-2节
        location: '教学楼A201',  // 可选
      }
    ],
    freeSlots: [
      { dayOfWeek: 3, periods: [5,6,7,8] },  // 周三5-8节空闲
      { dayOfWeek: 5, periods: [3,4,5,6,7,8] },
      ...
    ]
  },
  aiConfidence: number,         // AI识别置信度
  verified: boolean,            // 用户是否确认
  createdAt, updatedAt
}
```

## 五、三种排班模式详解

### 5.1 时间轴模式 (Timeline)
- **适用场景**：按具体时间段排班（如 8:00-10:00, 10:00-12:00）
- **模板特征**：每行是一个时间段，列是周一~周日
- **配置项**：开始时间、结束时间、是否包含午休、时段粒度

### 5.2 节次/班次模式 (Shift)
- **适用场景**：按课程节次排班（如 第1-2节, 第3-4节）
- **模板特征**：每行是一个节次，列是周一~周日
- **配置项**：总节次数、每节对应时间、是否包含午休

### 5.3 自定义模式 (Custom)
- **适用场景**：灵活的排班需求
- **自由定义**：行名称和数量、列名称和数量均可自定义
- **配置项**：完全自由的表格结构

## 八、排班算法设计（修正：服务端执行 + 随机抽取）

### 8.1 算法运行位置
- ❌ ~~客户端本地运算~~（原因：多端不一致、无法快速迭代、阻塞UI）
- ✅ 云函数服务端异步执行，客户端仅传约束参数

### 8.2 核心逻辑
- 每个时段从"标记为空闲"的人群中随机抽取 N 人
- N = `constraints.slot_min_people`（发布者在创建任务时设定，默认 1）
- 生成 3 套随机方案供发布者预览
- 发布者可手动调整任意时段的具体人选
- 右侧栏实时显示每人排班总次数/总时长

### 8.3 输出
- 3 套随机抽取方案（不做均衡/紧凑区分，纯随机）
- 发布者选定一套后 → 可手动微调具体人选 → 发布
- 发布后锁定，他人不可修改

## 九、技术架构

## 九、技术架构

### 前端（微信小程序）
- 框架: 原生微信小程序 (WXML + WXSS + JS)
- 状态管理: 全局 app.globalData + 页面级 data
- 网络请求: 统一封装的 wx.request Promise 层
- UI: TDesign 小程序组件库 + 自定义组件
- 排班算法: 不在客户端运行

### 后端（云函数 + CloudBase）
- 云函数: 排班方案生成（异步随机抽取算法）
- 数据库: CloudBase MySQL
- 定时任务: 截止时间回收（30s 扫描一次）
- 消息推送: 订阅消息（4 个模板）
- 存储: 云存储（图片/文件）

### H5 后台（运维端）
- 框架: React + Antd Pro + Vite
- 部署: CloudBase 静态托管
- 登录: 独立账号+密码（Superadmin 手动创建 Admin 账号）
- 功能: 可视化配置、数据大屏、审计日志、用户管理
