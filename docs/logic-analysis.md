# 排班小程序 — 完整业务逻辑分析

> 版本: v1.0 | 日期: 2026-07-03 | 基于 drawio 设计稿 + 需求文档

---

## 一、产品定位

**目标用户**：在校大学生值班排班场景（首期），后续可扩展至个人及企业。

**核心价值**：
- 管理员快速创建排班任务，通过微信邀请码分发
- 加入者像"共享表格"一样标记空闲时间
- 支持上传课表图片 AI 自动识别空闲时段
- 最终智能生成多种排班方案供管理员选择

## 二、角色与权限体系

### 2.1 两种角色

| 维度 | 管理员 (Admin) | 加入者 (Joiner) |
|------|---------------|----------------|
| 创建排班分组 | ✅ | ❌ |
| 定义节次/时间段参数 | ✅ | ❌ |
| 发布排班任务 | ✅ | ❌ |
| 查看联系人详情 | ✅ | 仅自己 |
| 选择排班方案 | ✅ | ❌ |
| 通过邀请码加入 | ❌ | ✅ |
| 标记空闲时间 | ✅ | ✅ (加入者主要操作) |
| 上传课表图片 | ✅ | ✅ |
| 查看自身日程 | ✅ | ✅ |
| 身份切换 | ✅ (可同时担任多个分组的 admin/joiner) |

### 2.2 身份切换机制

用户可以在不同排班分组中拥有不同身份：
- 在分组 A 中是管理员
- 在分组 B 中是加入者
- 通过顶部下拉或分组列表切换角色上下文

```javascript
// 用户-分组关系数据结构
{
  userId: "user_123",
  groups: [
    { groupId: "g1", role: "admin", groupName: "学生会值班" },
    { groupId: "g2", role: "joiner", groupName: "实验室值班" }
  ],
  activeGroupId: "g1"  // 当前活跃分组
}
```

## 三、核心数据模型

### 3.1 排班分组 (SchedulingGroup)

```
{
  id: string,
  name: string,
  inviteCode: string,        // 6位字母数字邀请码
  createdBy: userId,
  mode: 'timeline' | 'shift' | 'custom',  // 排班模式
  timeConfig: {              // 时间段/节次配置
    periods: [
      { id: 'p1', name: '第一节(8:00-9:40)', start: '08:00', end: '09:40' },
      { id: 'p2', name: '第二节(10:00-11:40)', start: '10:00', end: '11:40' },
      ...
    ],
    dateRange: { start: '2026-09-01', end: '2026-12-31' },
    weekDays: [1,2,3,4,5],    // 星期一~五
  },
  members: [
    { userId, name, avatar, phone, role: 'admin'|'joiner' }
  ],
  status: 'draft' | 'active' | 'completed' | 'archived',
  createdAt, updatedAt
}
```

### 3.2 排班任务 (SchedulingTask)

```
{
  id: string,
  groupId: string,
  title: string,
  description: string,
  dateRange: { start: '2026-09-18', end: '2026-09-24' },  // 本周/下周
  status: 'collecting' | 'completed',  // 正在排班 / 已排班
  template: {                    // 排班样式模板
    style: 1 | 2 | 3,
    periods: ['p1', 'p2', 'p3'], // 适用的节次
  },
  responses: [                   // 加入者的空闲时间标记
    {
      userId: string,
      availableSlots: [
        { date: '2026-09-18', periodId: 'p1' },
        ...
      ],
      source: 'manual' | 'ai_course_table',  // 来源
      submittedAt: timestamp,
    }
  ],
  finalSchedules: [              // 最终排班方案（多方案）
    {
      id: string,
      selected: boolean,
      assignments: [
        { date: '2026-09-18', periodId: 'p1', userId: string, userName: string }
      ]
    }
  ],
  createdAt, updatedAt
}
```

### 3.3 用户课表 (UserCourseTable)

```
{
  userId: string,
  imageUrl: string,            // 原始课表图片
  parsed: {
    semester: '2026-2027学年第1学期',
    courses: [
      {
        name: '高等数学',
        weeks: [1,2,3,...,16],  // 哪些周上课
        dayOfWeek: 1,            // 周一=1
        periods: [1,2],          // 第1-2节
        location: '教学楼A201',  // 可选
      }
    ],
    freeSlots: [
      { dayOfWeek: 3, periods: [5,6,7,8] },  // 周三5-8节空闲
      { dayOfWeek: 5, periods: [3,4,5,6,7,8] },
      ...
    ]
  },
  aiConfidence: number,         // AI识别置信度
  verified: boolean,            // 用户是否确认
  createdAt, updatedAt
}
```

## 四、完整用户流程

### 4.1 新用户首次进入

```
[打开小程序]
    │
    ├── 有邀请码 ──→ [点击"我有邀请码，我要加入"]
    │       │
    │       ├── 输入邀请码 → 验证
    │       │     ├── 成功 → 跳转首页（自动加入该分组）
    │       │     └── 失败 → 弹窗提示"邀请码错误，请重新输入"
    │       │
    ├── 无邀请码 ──→ [点击"创建分组"]
    │       │
    │       ├── 填写分组名称
    │       ├── 选择排班模式（时间轴/节次/自定义）
    │       ├── 配置时间段/节次参数
    │       └── 创建成功 → 生成邀请码 → 跳转首页
```

### 4.2 管理员完整操作流

```
[管理员首页]
    │
    ├── 查看已有分组
    │     └── 点击分组 → 进入该分组管理页
    │
    ├── 创建新排班任务
    │     ├── 选择模板样式(1/2/3)
    │     ├── 选择日期范围（本周/下周/自定义）
    │     ├── 选择适用节次/时间段
    │     ├── 设置任务标题、描述
    │     └── 发布任务 → 自动生成分享卡片
    │
    ├── 分享排班任务
    │     ├── 微信好友/群聊分享
    │     │     └── 卡片包含：分组名、任务标题、邀请码
    │     └── 复制邀请码
    │
    ├── 查看数据收集进度
    │     ├── 列表显示成员提交状态（已提交/未提交）
    │     └── 查看每个人的空闲时间段
    │
    └── 选择排班方案
          ├── 系统生成多种排班方案（算法自动分配）
          ├── 预览每种方案
          ├── 支持手动微调
          └── 确认发布 → 通知所有成员
```

### 4.3 加入者核心流程

```
[加入者首页]（通过邀请码加入后）
    │
    ├── 查看收到的排班任务
    │     ├── "正在排班" 列表
    │     └── "已排班" 列表
    │
    ├── 标记空闲时间（核心操作）
    │     ├── 方式一：手动标记
    │     │     ├── 显示7列（周一~周日）× 行（节次/时间段）表格
    │     │     ├── 点击格子切换"空闲/忙碌"状态
    │     │     └── 支持批量选择、拖选
    │     │
    │     ├── 方式二：AI课表识别
    │     │     ├── 上传课表截图/拍照
    │     │     ├── AI识别课程安排
    │     │     ├── 自动计算空闲时段
    │     │     ├── 展示识别结果 → 用户确认/修正
    │     │     └── 确认后自动应用到空闲标记
    │     │
    │     └── 二次筛选确认
    │           ├── 系统根据已标记空闲时间
    │           └── 与排班需求交叉筛选 → 确认可值班时间
    │
    └── 查看日程
          ├── 月份日历视图（7×5/6网格）
          ├── 查看自己已确定的排班
          └── 查看任务详情（时间、地点、联系人）
```

## 五、三种排班模式详解

### 5.1 时间轴模式 (Timeline)
- **适用场景**：按具体时间段排班（如 8:00-10:00, 10:00-12:00）
- **模板特征**：每行是一个时间段，列是周一~周日
- **配置项**：开始时间、结束时间、是否包含午休、时段粒度

### 5.2 节次/班次模式 (Shift)
- **适用场景**：按课程节次排班（如 第1-2节, 第3-4节）
- **模板特征**：每行是一个节次，列是周一~周日
- **配置项**：总节次数、每节对应时间、是否包含午休

### 5.3 自定义模式 (Custom)
- **适用场景**：灵活的排班需求
- **自由定义**：行名称和数量、列名称和数量均可自定义
- **配置项**：完全自由的表格结构

## 六、页面状态机详解

### 6.1 首页 (Home Page)

```
状态:
  LOADING           → 加载中 (骨架屏)
  NEW_USER          → 首次用户（2个入口按钮 + 3种模式卡片）
  HAS_GROUPS        → 已有分组用户
  └── admin_view    → 管理员视角（创建任务入口 + 分组管理）
  └── joiner_view   → 加入者视角（待标记任务 + 日程总览）

事件:
  onTapCreateGroup      → 跳转创建分组页面
  onTapJoinByCode       → 弹出邀请码输入框
  onTapModeCard(mode)   → 选择排班模式，创建新分组
  onTapGroup(groupId)   → 进入分组详情
  onTapTask(taskId)     → 进入任务详情/标记页
```

### 6.2 日程页 (Schedule Page)

```
状态:
  LOADING           → 加载中
  CALENDAR_VIEW     → 日历主视图
  TASK_LIST         → 任务列表
  TASK_DETAIL       → 任务详情弹窗
  EMPTY             → 暂无排班数据

事件:
  onMonthChange     → 切换月份
  onDateTap(date)   → 点击日期查看当天排班
    └── 弹出详情: 名字、地点、时间、联系方式
  onTaskTap(taskId) → 查看任务详情
  onTabSwitch       → 切换"正在排班"/"已排班"

数据结构:
  calendarDays: [{date, assignments: [{user, period, location}]}]
  activeTasks: [SchedulingTask]     // status='collecting'
  completedTasks: [SchedulingTask]  // status='completed'
```

### 6.3 加入/编辑页 (Join/Edit Page)

```
状态:
  INITIAL           → 初始：两个入口按钮
  INVITE_INPUT      → 邀请码输入界面 
  INVITE_LOADING    → 验证中
  INVITE_SUCCESS    → 验证成功，自动跳转
  INVITE_FAILED     → 显示"邀请码错误"
  CREATE_GROUP      → 创建分组表单
  CREATE_SUCCESS    → 创建成功，展示邀请码

流程:
  [初始页]
    ├── "我有邀请码，我要加入" → [输入邀请码] → [验证] → 成功/失败
    └── "创建分组" → [表单页] → [生成邀请码] → [分享]
```

### 6.4 模板页 (Template Page)

```
状态:
  TEMPLATE_LIST     → 三种样式预览
  TEMPLATE_EDIT     → 编辑模板参数
  PERIOD_CONFIG     → 节次/时间段配置
  AI_UPLOAD         → 上传课程表图片

三种样式:
  样式1: 时间轴样式 (横向时间段)
  样式2: 节次样式 (课程节次)
  样式3: 自定义样式 (灵活表格)

核心交互:
  - 点击样式卡片 → 预览 + 编辑参数
  - 配置节次：添加/删除/修改节次名称和时间
  - 配置午休：开关 + 时间段设置
  - AI识别入口 → 跳转AI帮识别页
```

### 6.5 AI帮识别页 (AI Recognition Page)

```
状态:
  PLACEHOLDER       → "正在开发中" 占位页（当前版本）
  UPLOAD            → 图片上传
  UPLOADING         → 上传中
  RECOGNIZING       → AI识别中
  RESULT_PREVIEW    → 识别结果预览
    ├── 时间段维度
    ├── 节次/课次维度
    └── 周次维度
  CONFIRMED         → 确认并应用

（首期 MVP 为占位页，后续迭代实现完整功能）
```

## 七、邀请码系统设计

### 7.1 邀请码生成规则
- 格式: 6位大写字母+数字组合 (如 A3X9K2)
- 生成方式: 随机生成 + 数据库唯一性校验
- 有效期: 永久有效(或可设置过期时间)
- 一个分组一个邀请码

### 7.2 邀请码使用流程
```
[管理员] 生成邀请码 → 分享到微信群/好友
    ↓
[加入者] 输入邀请码 → 后端验证 → 返回分组信息
    ↓
[加入者] 确认加入 → 自动绑定到分组 → 跳转首页
```

## 八、排班算法初步设计

### 8.1 输入
- N 个成员 × M 个时间段(日期×节次) 的空闲矩阵
- 每个时段需要的值班人数 K

### 8.2 约束条件
1. 每人每周最多值班次数限制
2. 每人每天最多值班时段限制
3. 尽量均匀分配（公平性）
4. 连续排班避免

### 8.3 输出
- 多种排班方案（如均衡型、紧凑型、偏好优先型）
- 每个方案评分（公平性得分、覆盖度得分）

### 8.4 首期简化策略
MVP 阶段采用贪心算法 + 启发式规则，后续迭代引入更复杂的约束求解。
